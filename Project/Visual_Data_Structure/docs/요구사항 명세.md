# 요구사항 명세
VDS 간단한 요구사항 명세를 통해서 구현 기준점을 잡으려고 함.

## 유스케이스
### A. 실행/시뮬레이션(Runner)

- UC-RUN-001: 시뮬레이터가 시나리오(연산 시퀀스)를 실행한다
- UC-RUN-002: GUI 사용자가 연산 1개를 요청한다(push/pop/top/clear)
- UC-RUN-003: 연산 1개 실행 후 Scene 프레임 1개를 반환/발행한다
- UC-RUN-004: N-step 실행(일괄 실행) 후 Scene 프레임 리스트를 반환한다
- UC-RUN-005: 이전 실행 기록(연산 로그/프레임)을 리플레이한다


### B. Core(도메인)
**B-1 Stack 연산**
- UC-STK-001: push(value) 한다
- UC-STK-002: pop() 한다 (최상단 제거 + 반환)
- UC-STK-003: top() 한다 (제거 없이 조회)
- UC-STK-004: is_empty()를 조회한다
- UC-STK-005: size()를 조회한다
- UC-STK-006: clear()로 모두 비운다
- UC-STK-007: snapshot()으로 현재 요소 목록을 안전 복사로 얻는다(표현용)

### C. Presenter - Scene 생성/표현 규칙

- UC-PRS-001: 현재 Stack 상태를 Scene으로 변환한다(정적 프레임)
- UC-PRS-002: 연산(push/pop/top/clear) 결과를 반영한 Scene 프레임을 만든다
- UC-PRS-003: 요소 role/style 규칙에 따라 Rect/Text 스타일을 매핑한다
- UC-PRS-004: 키(key) 규칙으로 Drawable의 안정적 식별자를 부한다(애니메이션/보간 대비)
- UC-PRS-005: Scene의 메타(title/complexity/설명) 정보를 포함한다
- UC-PRS-006: Scene 크기/레이아웃(패딩, 간격, 정렬)을 규칙 기반으로 계산한다

### D. 에러/경계 조건

- UC-ERR-001: 빈 스택 pop/top 요청 시 EmptyStackError를 발생시키고, Presenter는 이를 오류 프레임(Scene) 또는 오류 상태로 표현한다
- UC-ERR-002: 입력값 검증(예: None/지원하지 않는 타입/너무 긴 문자열 등) 실패 시 **ValidationError(또는 DomainError)**로 처리한다
