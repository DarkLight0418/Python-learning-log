# Python TCP Socket GUI Chat Application

Python의 TCP Socket과 wxPython을 활용하여 구현한 GUI 기반 실시간 채팅 프로그램입니다.  
서버 1개와 여러 클라이언트가 연결되는 클라이언트-서버 구조로 동작하며, 각 클라이언트는 GUI 화면을 통해 메시지를 주고받을 수 있습니다.

## 1. 프로젝트 개요

이 프로젝트는 네트워크 소켓 프로그래밍 학습을 목적으로 제작한 채팅 애플리케이션입니다.

주요 목표는 다음과 같습니다.

- TCP Socket 기반 클라이언트-서버 통신 구현
- 여러 클라이언트의 동시 접속 처리
- wxPython을 이용한 GUI 채팅 화면 구현
- JSON 기반 메시지 프로토콜 설계
- GUI, 네트워크, 서버, 프로토콜 책임 분리

## 2. 기술 스택

| 구분 | 사용 기술 |
|---|---|
| Language | Python |
| Network | socket |
| Concurrency | threading |
| GUI | wxPython |
| Message Format | JSON |
| Message Delimiter | Newline `\n` |

## 3. 프로젝트 구조

```text
chat_app/
├── server.py
├── client_gui_wx.py
├── client_network.py
└── protocol.py
```

| 파일 | 역할 |
|---|---|
| `server.py` | 서버 실행, 클라이언트 연결 관리, 메시지 브로드캐스트 |
| `client_gui_wx.py` | wxPython 기반 GUI 화면 및 사용자 이벤트 처리 |
| `client_network.py` | 서버 연결, 메시지 송신/수신 처리 |
| `protocol.py` | 메시지 생성, 인코딩, 디코딩, 분리 처리 |

## 4. 실행 방법

### 4.1 패키지 설치

```bash
pip install wxPython
```

### 4.2 서버 실행

```bash
python server.py
```

서버는 기본적으로 다음 주소에서 실행됩니다.

```text
HOST = 0.0.0.0
PORT = 5000
```

### 4.3 클라이언트 실행

```bash
python client_gui_wx.py
```

같은 PC에서 테스트할 경우 클라이언트 접속 정보는 다음과 같이 입력합니다.

```text
Server IP: 127.0.0.1
Port: 5000
Nickname: 원하는 닉네임
```

클라이언트를 2개 이상 실행하면 서로 메시지를 주고받을 수 있습니다.

## 5. 동작 흐름

### 5.1 서버 흐름

```text
서버 실행
↓
클라이언트 접속 대기
↓
클라이언트 연결 수락
↓
클라이언트별 수신 스레드 생성
↓
메시지 수신
↓
전체 클라이언트에게 브로드캐스트
```

### 5.2 클라이언트 흐름

```text
GUI에서 서버 IP, 포트, 닉네임 입력
↓
Connect 버튼 클릭
↓
서버 연결
↓
메시지 입력 후 Send 또는 Enter
↓
서버로 메시지 전송
↓
서버에서 받은 메시지를 GUI에 출력
```

## 6. 메시지 프로토콜

메시지는 JSON 형식으로 구성되며, TCP 스트림에서 메시지를 구분하기 위해 각 메시지 끝에 줄바꿈 문자 `\n`을 붙입니다.

### 채팅 메시지 예시

```json
{
  "type": "chat",
  "sender": "hanjae",
  "message": "안녕하세요"
}
```

### 입장 메시지 예시

```json
{
  "type": "join",
  "sender": "hanjae",
  "message": ""
}
```

### 퇴장 메시지 예시

```json
{
  "type": "leave",
  "sender": "hanjae",
  "message": ""
}
```

## 7. 주요 설계 특징

### 7.1 책임 분리

이 프로젝트는 파일별 책임을 분리하여 설계했습니다.

```text
GUI는 화면과 사용자 이벤트만 담당합니다.
Network는 서버와의 송수신만 담당합니다.
Server는 클라이언트 연결 관리와 메시지 중계만 담당합니다.
Protocol은 메시지 형식 변환만 담당합니다.
```

이를 통해 GUI 코드가 소켓 처리 로직에 직접 의존하지 않도록 구성했습니다.

### 7.2 Queue 기반 GUI 갱신

네트워크 수신 작업은 별도 스레드에서 실행됩니다.  
wxPython GUI 위젯은 메인 스레드에서만 안전하게 수정할 수 있으므로, 수신 스레드는 메시지를 직접 화면에 출력하지 않고 Queue에 넣습니다.

이후 GUI는 `wx.Timer`를 통해 Queue를 주기적으로 확인하고 메시지를 화면에 출력합니다.

```text
수신 스레드
↓
Queue에 메시지 저장
↓
wx.Timer가 Queue 확인
↓
GUI 메인 스레드에서 채팅창 갱신
```

### 7.3 멀티 클라이언트 처리

서버는 클라이언트가 접속할 때마다 별도의 스레드를 생성합니다.  
각 클라이언트 스레드는 해당 클라이언트의 메시지를 수신하고, 서버는 받은 메시지를 전체 클라이언트에게 전달합니다.

## 8. 테스트 방법

1. 서버를 먼저 실행합니다.

```bash
python server.py
```

2. 클라이언트를 2개 이상 실행합니다.

```bash
python client_gui_wx.py
```

3. 각 클라이언트에서 다음 정보로 접속합니다.

```text
Server IP: 127.0.0.1
Port: 5000
Nickname: 사용자별 닉네임
```

4. 한 클라이언트에서 메시지를 보내고, 다른 클라이언트 화면에 출력되는지 확인합니다.

## 9. 다른 컴퓨터에서 접속하기

같은 와이파이에 연결된 다른 컴퓨터에서 접속하려면 클라이언트의 Server IP에 서버 컴퓨터의 IPv4 주소를 입력합니다.

예시:

```text
서버 PC IPv4 주소: 192.168.0.15
클라이언트 Server IP: 192.168.0.15
Port: 5000
```

접속이 되지 않을 경우 다음 항목을 확인합니다.

- 서버와 클라이언트가 같은 네트워크에 연결되어 있는지 확인
- 서버가 `0.0.0.0`으로 실행되고 있는지 확인
- Windows 방화벽에서 Python 실행이 허용되어 있는지 확인
- 포트 번호가 서버와 클라이언트에서 동일한지 확인
- 공용 와이파이에서 기기 간 통신이 차단되지 않았는지 확인
